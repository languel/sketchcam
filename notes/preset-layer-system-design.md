# Layer + Routing System — Design

Branch: `presets` · Status: **draft for review** (design-doc-first; no code yet)

## Vision
A **routing/patch graph** for SketchCam. Two kinds of signal flow through it:

- **Pixels / textures** (raster): camera, images, movies, web, or generated by a
  process (ink sim output, effect output, …).
- **Paths** (vector/points): feature analysis (landmarks), brush strokes (mouse),
  and other inputs later (OSC, network, multitouch).

Processing units transform these signals:

- **Sources** — produce a signal (camera→pixels, landmarks→paths, mouse→paths,
  web→pixels).
- **Drawing algorithms** — analysis/paths → new paths (yarn, wrap, lineWalk).
- **Brushes** — paths → pixels (how a path is *drawn*: pen ribbon, wash, …).
- **Effects** — pixels → pixels (threshold, outline, blur).

**Layers** are the pixel-producing nodes that get **composited** into the final
frame (ordered stack, blend/opacity/mask). The point of the system is **routing**:
any node's input can be bound to a shared source *or to another node's output* —
e.g. the **web layer's output becomes the texture input of the ink layer**. So
under the hood it's a small **DAG of typed nodes**; the layer stack is the ordered
subset of pixel nodes that are composited.

> Build order the user set: **layers first, routing next, presets once features
> settle.** Unique instances for now (one ink, one web) — but model ports so
> multiplicity and routing drop in later without a rewrite.

## Where we are today
`ProcessingSettings` holds every feature's config; `CoreImageFrameProcessor`
composites camera → effects → background → person-key → drawing → ink → web in a
**fixed hardcoded order** with a couple of placement enums. No stack, no ordering,
no routing, no reuse.

## Model (Core, Codable)

```
// A typed signal a port carries.
enum SignalType { case pixel, path }

// A node = one processing unit with typed input ports and one output.
struct Node: Identifiable, Codable, Equatable {
    var id: UUID
    var name: String
    var kind: NodeKind            // payload + declares its ports + output type
    var inputs: [PortBinding]     // one per declared input port
}

// Where an input port gets its signal from.
enum PortBinding: Codable, Equatable {
    case none                     // use the kind's built-in default
    case source(SourceID)         // a shared source (camera, landmarks, mouse…)
    case node(UUID)               // another node's output (the routing!)
}

enum NodeKind: Codable, Equatable {
    case video(VideoConfig)       // pixel out; in: source(pixel)
    case solid(SolidConfig)       // pixel out; no in
    case effect(EffectConfig)     // pixel→pixel; in: image(pixel)
    case marks(MarksConfig)       // pixel out; in: analysis(path)
    case drawing(DrawingConfig)   // ONE algorithm; in: analysis(path); out: pixel (via its brush)
    case ink(InkConfig)           // pixel out; in: strokes(path), texture(pixel, optional)
    case web(WebConfig)           // pixel out; no in (produces a texture)
    // future: image, brush-as-node, path-generators, osc, …
}

// The composite stack: which pixel nodes are layers, in what order, how blended.
struct Layer: Identifiable, Codable, Equatable {
    var id: UUID
    var node: UUID                // the pixel node this layer displays
    var visible: Bool
    var opacity: Float
    var blend: BlendMode          // design for ALL; implement incrementally
    var mask: LayerMask?          // .person(invert:) keys the layer to the subject
}

struct Document: Codable, Equatable {
    var version: Int
    var nodes: [Node]
    var layers: [Layer]           // ordered bottom→top; references nodes by id
}
```

Existing per-feature config (ink, drawing palettes, effects, web) **re-homes** into
the `*Config` payloads — mostly a move. Shared-input config (camera, detection
rate, segmentation quality, processing res, preview) stays on `ProcessingSettings`
alongside the `Document`.

### Ports & signal-type checking
Each `NodeKind` declares its input ports (name + `SignalType`) and its output type.
A `PortBinding.node(x)` is only valid if node x's output type matches the port's
type. Cycles are rejected (DAG). For now ports default to the built-in source
(e.g. ink.strokes ← mouse+drawing, ink.texture ← none) so phase 1 behaves exactly
like today without any explicit wiring.

### Masking replaces the standalone person-key
Person keying becomes a per-layer `mask: .person(invert:)` — any layer clips to the
subject; folds the segmentation matte in cleanly.

## Routing — the north star, proven incrementally
- Default (phase 1): every port uses its built-in source; layers composite
  bottom→top. Identical output to today.
- First real route (phase 2): expose **ink.texture** and let it bind to the **web
  node's output** (web→ink). One concrete, high-value wire that exercises the whole
  mechanism (typed port, node→node binding, render-order/dependency resolution).
- Then generalize: any pixel port can bind to any pixel node; path ports to path
  nodes; add path sources (OSC/multitouch) and path-generator nodes.

Render scheduling: topologically sort nodes by their bindings each frame (cheap,
≪ dozens of nodes), render once each, then composite the `layers`. Shared analysis
(detection, matte) stays computed once/frame, off-path, as it is now.

## Compositor refactor
Replace the fixed `process(...)` body: build shared context → topo-sort & render
nodes → composite `document.layers` (skip hidden / opacity≈0) with mask + blend +
opacity. Existing render code is reused per kind; the change is **iteration +
dependency order** instead of a hardcoded sequence. Per-layer stage timings in the
HUD.

## Migration / back-compat
Loading an old `ProcessingSettings` (no `Document`) synthesizes the default graph:
video(+effect) → marks → drawing (yarn/wrap/lineWalk as **separate** drawing nodes)
→ ink → web, honoring today's placement flags; person-key → masks. Versioned;
old presets migrate the same way.

## Presets (deferred — after features settle)
When ready: **global** = snapshot `Document` + shared config; **per-layer/per-node**
= snapshot one node's config (+ its layer's blend/opacity/mask), re-applied to a
same-kind node. **JSON files in Application Support** (exportable/shareable), not
UserDefaults. One Codable model, two entry points.

## UI (phased)
- Phase 1: Layers tab → real layer list (reorder, eye, opacity, blend menu, mask,
  add ＋ kind / duplicate / delete); selected layer reveals its existing controls
  inline.
- Later: per-port routing UI (a small dropdown per input port: "source ▸ …" /
  "layer ▸ …"), then optionally a node-graph view if it earns its keep.

## Decisions locked in (from review)
- **One algorithm per layer** (max flexibility; stack two lineWalks w/ different seeds).
- **Unique instances for now** (one web, one ink); model supports many later.
- **All blend modes** are the goal; implement incrementally (normal/multiply/screen first).
- **JSON** preset storage.
- Routing is the priority; presets deferred.

## Phasing (each phase shippable)
1. **Graph model + migration** (Core): `Node`/`Port`/`Layer`/`Document`, re-home
   configs, synthesize default graph from old settings, version + migrate. No
   behaviour change.
2. **Compositor refactor**: render nodes by dependency order + composite the layer
   stack; **must be pixel-identical** to today for the default graph (snapshot test).
3. **Layers UI**: reorder / visibility / opacity / blend(min) / mask / add-remove-
   duplicate, per-layer controls inline.
4. **Routing v1**: typed input ports + bind ink.texture ← web (the proof), with a
   per-port picker; topo-sort scheduling.
5. **Generalize**: more ports/routes, path signals & sources (OSC/multitouch),
   path-generator/brush-as-node, multiplicity, full blend modes.
6. **Presets** (global + per-node, JSON).

## v2 — the all-layers model (CHOSEN, supersedes the tracks above)

The earlier phasing wired a Layers panel that still *mirrored* feature flags, and a
graph compositor that only re-ordered the *movable* overlays over a hardcoded
camera/effects/background base. That's why the Camera "layer" is cosmetic (its
eye/opacity/order do nothing) — it's the base, not a layer. The agreed end-state
removes ALL hardcoded layering:

**Gone:** ink placement (behind/above), web placement (behind/above), `backgroundMode`
(live/solid/alpha), the standalone person-key, and the global camera/movie source
picker. Stacking order is *only* the layer stack.

**Every layer = content (a *stream*) + a mask + an effect chain.**

- **Streams (content kinds):** `camera`, `movie`, `solid` (with/without alpha),
  `drawing`, `web`, `ink`, `paper`. (`paper` is its own stream so ink can be drawn
  over the live camera — ink's substrate is a routed input, not a fixed background.)
- **Person matte is a stream**, selectable as any layer's mask source — not a toggle.
- **Routing via in-panel dropdowns.** A stream that needs another stream's output
  picks its source there: drawing ← landmark source; ink ← paper/substrate stream;
  later web → ink.texture. These are the `PortBinding.node` edges already in the model.
- **Mask dropdown at the top of every stream panel:** `NONE` or any other stream,
  with an optional **threshold / inverted-threshold** mode (luma-key the chosen
  stream into a matte).
- **Effects are an ordered chain *per layer*** (decision: chain, not a single slot).
  The layer mask clips the raw source first; the ordered effect chain then runs on
  that masked result; final blend/opacity happen only at composite time.
- **Video: separate `camera` and `movie` streams** (decision), both available at once
  → two simultaneous capture sources (capture-layer change, phase G4).

### Model deltas from v1
- `Layer` gains `effects: [EffectConfig]` (ordered) and a generalized
  `mask: MaskBinding?` = `{ source: StreamRef, mode: .luma | .threshold(level) | .invThreshold(level) }`;
  `.person(invert:)` becomes `mask.source = .stream(personMatteNode)`.
- `NodeKind` gains `camera`, `movie`, `paper`, `personMatte`; `solid` already carries
  alpha. `effect` as a standalone node kind is retired in favour of per-layer chains.
- Streams needing routed input expose typed input ports (already modelled);
  the panel binds them via `PortBinding.node`.

### Phasing (each shippable; parity-guarded; legacy stays behind `useLayerGraph` until parity)
- **G1 — Core model.** Evolve `Layer` (effects chain + generalized mask), add the new
  stream kinds, migration legacy→graph, routing source bindings. Renderers still read
  legacy → no behaviour change. Tests + Codable round-trip.
- **G2 — unified compositor. DONE (GPU, behind `useGPUCompositor`).** `MetalLayerCompositor`
  renders EVERY visible layer from the graph on the GPU — raw stream → layer mask →
  per-layer Metal effect chain (threshold/outline/blur/personKey) → source-over with
  opacity/blend — no hardcoded base.
  Keying = the camera layer's personMatte mask. defaultGraph re-homes legacy threshold/
  outline onto the camera chain; reconcile refreshes managed layers' kind/effects/mask
  from settings so the legacy tabs still drive it until G3. Verified live (CUA): raw
  camera as a layer, threshold+outline chain on GPU, hide-camera → transparent, Drawing
  composited over Camera. Effects move off CoreImage (CI stays as fallback). Remaining:
  blend modes (only normal now), GPU-native producers (overlay/ink/web still rasterized
  via CI each frame), perf tuning, and a pixel-parity test for the GPU path.
- **G3 — UI. DONE (mostly).** Per-layer effect-chain editor (Blender-modifier-style
  collapsible panels: add/reorder/enable/delete; threshold/outline/blur/invert/mirror/
  personKey). Legacy Effect + Background tabs removed. Person key is a per-layer effect
  (with Silhouette flat-fill + matte Quality in-panel). Layers are renamable (double-
  click). Add-layer menu offers Camera/Movie/Solid/Paper + Drawing/Ink/Web. Camera &
  Movie are now separate tabs (combined source toggle gone).
  **Mask dropdown DONE:** each expanded layer has a Mask control (None / Person matte /
  any named stream) + mode (Luma/Threshold/Inv) + level + invert; compositor applies it.
  STILL TODO in G3: **routing source dropdowns** for *producer* inputs (ink.texture ← a
  stream, drawing ← landmark source). Harder than masks — producers (ink/drawing) are
  computed in the VM, so the compositor must resolve a `.node(id)` pixel input and feed
  it to the producer, not just read another layer's image. Deferred.

## Freeform multiplicity track (the Photoshop goal) — chosen direction

Phase 3a wired a Layers panel, but layers still *mirror the feature flags* (one of
each kind), so it doesn't feel freeform. Target end-state:

- **The graph is the sole source of truth.** Every layer is a node with its OWN
  config; no feature flags driving which layers exist.
- **The compositor builds the whole frame by iterating layers** bottom→top —
  including camera / effects / background / person-key as layer *kinds*, with no
  special-cased "base". Arbitrary order (e.g. a solid below the camera) just works.
- **Renderers become per-layer instances**: one ink fluid-sim engine per ink
  layer, one WKWebView per web layer, per-algorithm drawing, cheap video/solid/
  image. Multiple of any kind allowed.
- **Panel owns layers**: ＋Add (kind menu), delete, duplicate, reorder, and
  select-a-layer-to-edit (the per-kind controls move under the selected layer).
  Per layer: visible, opacity, blend (all modes), mask. Legacy feature tabs are
  subsumed.

### Phasing (each shippable; parity-guarded where it can regress)
- **F1 — per-node config (Core).** Re-home each feature's config into the node
  payloads (Video/Solid/Effect/Overlay(Drawing)/Ink/Web configs). Migration
  populates them from the legacy flat settings. `Node.managed` distinguishes
  feature-derived vs user-added layers (reconciliation preserves user-added).
  Renderers still read legacy → no behavior change. Tested.
- **F2 — unify the compositor.** Render EVERY layer from the graph (incl.
  camera/effects/bg/key); drop the special base. Must be pixel-identical for the
  default graph (extend the parity test). Unlocks arbitrary ordering.
- **F3 — multiplicity for cheap kinds + panel add/remove.** Solid, image, and
  per-algorithm drawing become freely add/duplicate/delete/reorder from the panel
  (instances + per-node config + N-compositing). First visible freeform layers.
- **F4 — multiplicity for heavy kinds.** N ink engines / N web views, with cost
  warnings (you noted you won't run many at once).
- **F5 — panel polish.** Blend-mode menu (all modes), mask UI, thumbnails,
  select-to-edit; retire the legacy feature toggles.
- Then the deferred tracks: **routing** (ink.texture ← web) and **presets**
  (global + per-node, JSON).

Biggest risk = F2 (the keying/silhouette/effects logic is delicate) → pixel
snapshot parity before/after. Heavy-kind multiplicity (F4) is perf-bounded.

## Risks / open questions
- Compositor refactor must be output-identical (regression surface) → pixel
  snapshot test on the default graph before/after.
- Routing introduces dependency ordering + signal-type validation + cycle
  rejection — keep it tiny and explicit; don't build a generic graph engine before
  one real route works.
- **Q:** for phase 1, keep the current Layers tab semantics and grow it, or design
  the new layer list UI fresh? (Leaning: grow it.)
- **Q:** path signals — in scope to *route* this cycle, or just reserve the type in
  the model and wire only pixel routing first? (Leaning: reserve type, pixel
  routing first.)
